# T0 Pulse | T+0盘中脉冲监控系统
## 🎯 产品定位
**持仓的"盘中护航仪"**
4个关键时间节点（10:00 / 11:00 / 13:30 / 14:30）自动推送持仓诊断和T+0操作信号，让持仓过夜的上班族也能系统化管理仓位。
## ⏰ 四个监控节点
| 时间 | 节点 | 重点任务 |
|------|------|---------|
| **10:00** | 开盘确认 | 开盘第一波方向判断，是上还是下 |
| **11:00** | 午盘前决策 | 反弹减仓还是回调加仓 |
| **13:30** | 下午方向 | 午后第一波方向确认 |
| **14:30** | 尾盘行动 | 留仓/清仓/做T决策 |
## 📊 信号体系（五色预警）
| 信号 | 颜色 | 条件 | 操作建议 |
|------|------|------|---------|
| **超跌** | 🔴 | RSI<30 或 BB<15% 或 跌幅>2% | 可小仓补/做T降成本 |
| **偏弱** | 🔴 | 跌幅1-2% | 持有，不加仓 |
| **震荡** | ⚪ | 涨跌互现，幅度<1% | 观望，不操作 |
| **偏强** | 🟡 | 涨幅1-2% | 持有，等情绪延续 |
| **超涨** | 🟢 | RSI>70 或 BB>85% 或 涨幅>3% | 可减仓，锁定利润 |
## 🛠️ 核心功能
### 1. 实时持仓诊断
- 读取持仓列表（支持多账户）
- 获取实时行情（价格、涨跌幅、RSI、BB%、量比）
- 五色信号实时判定
### 2. 超跌超涨预警

### 3. T+0操作建议
内置两条核心原则：
- **恐慌杀跌时不追：** 等5-10分钟反弹再出，不要开盘立刻割
- **每日目标2个点：** 每只票每天做2个点T，月化40%+
## 📈 内置T+0操作原则
### 买进策略（加仓/建仓）
- 超跌信号出现后，等价格企稳再买
- 不要在恐慌杀跌时接飞刀
- 买入后等反弹3-5分钟即可卖出
### 卖出策略（减仓/止损）
- 超涨信号出现后，分批减仓
- 不要贪心到最高点才卖
- 每次卖出1/3～1/2仓位
### 日内T+0降成本
示例（润泽科技，成本89.92）：
开盘下跌出现超跌信号
在81.5附近小仓补进1000股
反弹到83.0附近卖出这1000股
赚取1.5元/股 = 1500元降成本
## ⚙️ 配置说明
### 1. 数据源配置
使用 codebuddy.cn 行情API（需注册获取Token）
### 2. 飞书Webhook配置
在脚本中配置 `FEISHU_WEBHOOK`
### 3. 持仓配置
编辑 `holdings_config.py` 中的 `HOLD_STOCKS` 字典
### 4. 持仓更新
盘中换仓后立即更新配置文件，下一个推送自动使用新持仓
## 💡 使用场景
- 上班族开盘后设置提醒，10点/11点/13:30/14:30看诊断
- 持仓亏损时判断是持有还是止损
- 想做T降成本但没时间盯盘
- 尾盘不知道留仓还是清仓
## ⚠️ 风险提示
- T+0操作要求执行力强，严格遵守止损
- 信号出现不等于一定会反弹，仍需结合大盘判断
- 建议仓位不超过总资金30%时做T，避免加仓扩大风险
